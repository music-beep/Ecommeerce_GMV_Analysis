# Ecommeerce_GMV_Analysis
# 电商用户行为分析与 GMV 优化项目 (E-commerce User Behavior & GMV Optimization)

## 项目链接：https://tianchi.aliyun.com/dataset/154063

## 📌 项目背景
本项目基于某电商平台的 10 万条用户行为数据，旨在通过数据分析拆解 GMV（商品交易总额），寻找影响用户转化的关键因素。
项目核心目标遵循公式：**GMV = UV (独立访客) * 转化率 * 客单价**。
由于数据集中不包含客单价信息，本分析重点在于**提升转化率**和**挖掘高价值用户特征**。

## 🛠️ 技术栈
* **语言:** Python
* **数据处理:** Pandas, NumPy
* **可视化:** Matplotlib, Seaborn, Plotly (漏斗图)
* **机器学习:** Scikit-learn (Random Forest, Pipeline, OneHotEncoder)

## 📊 数据概览
数据集包含 100,000 条用户记录，涵盖以下关键字段:
* **用户属性:** 年龄 (Age), 性别 (Sex), 市场级别 (Market), 用户来源 (Source), 是否新用户 (New User).
* **行为数据:** 浏览页面总数 (Total Pages), 浏览主页/列表页/详情页/支付页/确认页 (Funnel Steps).
* **设备信息:** 设备类型 (Device), 操作系统 (OS).

## 💡 核心分析过程

### 1. 数据预处理
* **异常值处理:** 发现年龄字段存在最大值为 123 岁的异常数据，剔除了 >80 岁的记录.
* **缺失值填充:** 对性别、设备等分类变量使用众数填充，年龄使用均值填充.
* **特征工程:** 将连续的年龄变量离散化为年龄段 (bins: 18-25, 26-35 等)，以便进行分群分析.

### 2. 漏斗分析 (Funnel Analysis)
构建了完整的用户转化漏斗：`主页 -> 列表页 -> 详情页 -> 支付页 -> 支付完成`。
* **发现:** 用户从 **"详情页" 到 "支付页"** 的流失最为严重（从 49.6k 跌至 6.8k），这不仅是技术瓶颈，更是运营转化的关键节点.

### 3. 用户画像与转化率分析
通过多维度可视化分析发现：
* **来源渠道:** `Direct` (直接访问) 的转化率最高 (3.03%)，其次是 `Ads` (2.60%)，`SEO` 最低.
* **设备差异:** `Desktop` (桌面端) 转化率 (2.60%) 略高于 `Mobile` (移动端) (2.27%)，提示移动端体验有优化空间.
* **年龄分布:** 年轻群体（18-25岁, 26-35岁）是消费主力，转化率随年龄增长而下降.

### 4. 机器学习归因分析 (Random Forest)
使用随机森林模型预测用户是否会完成支付。
* **防泄漏处理:** 严格剔除了 `浏览过支付页` 和 `支付完成页` 本身，防止数据穿越.
* **特征重要性结论:**
    1.  **浏览页面总数 (0.5007):** 用户活跃度是预测转化的最强指标。
    2.  **年龄 (0.2615):** 最关键的人口统计学特征，甚至超过了市场级别 (0.0628)。
    3.  **浏览过详情页 (0.0669):** 再次印证了详情页是转化的核心环节。

## 🚀 商业建议 (Actionable Insights)
基于分析结果，提出以下 GMV 提升策略：

1.  **聚焦关键人群:** 重点投放和运营 **18-35 岁** 的年轻用户群体，这部分人群贡献了最高的模型权重和实际转化率。
2.  **详情页优化:** 针对“详情页 -> 支付页”的高流失率，建议优化商品详情页的 UI/UX 设计，增加“立即购买”的引导强度，并检查页面加载性能。
3.  **渠道策略:** `Direct` 流量质量最高，建议通过会员体系或订阅服务增加用户粘性；`Ads` 转化尚可，可继续投放；需重新评估 `SEO` 策略的关键词精准度。
4.  **移动端体验:** 鉴于移动端转化率低于桌面端，建议技术团队进行移动端适配性测试，消除支付链路中的摩擦。
